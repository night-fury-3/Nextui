# Colors

The `nextui` plugin allows you to customize the semantic colors of the theme, for example, the `primary`,
`secondary`, `success`, etc...

```js {7,11}
module.exports = {
  plugins: [
    nextui({
      themes: {
        light: {
          // ...
          colors: {},
        },
        dark: {
          // ...
          colors: {},
        },
        // ... custom themes
      },
    }),
  ],
};
```

> Colors options are applied to all components.

## Default Colors

NextUI includes a default color palette out-of-the-box that is a great starting point if you don't have your
own specific branding in mind.

They are separated into [Common Colors](#common-colors) and [Semantic Colors](#semantic-colors).

- [Common Colors](#common-colors) colors that are always the same, they don't change with the theme.
- [Semantic Colors](#semantic-colors) colors that change with the theme.

### Common Colors

You can see the common colors as the [TailwindCSS](https://tailwindcss.com/docs/customizing-colors) colors
they don't change with the theme, they are always the same. They extend the [TailwindCSS](https://tailwindcss.com/docs/customizing-colors) colors.

Common colors are omitted by default to avoid conflicts with the [TailwindCSS](https://tailwindcss.com/docs/customizing-colors) colors.
You can enable them with the `addCommonColors` option.

```js {4}
module.exports = {
  plugins: [
    nextui({
      addCommonColors: true,
    }),
  ],
};
```

It will add the following colors to the [TailwindCSS](https://tailwindcss.com/docs/customizing-colors) colors.

```js
module.exports = {
  theme: {
    extend: {
      colors: {
        white: "#FFFFFF",
        black: "#000000",
        blue: {
          50: "#e6f1fe",
          100: "#cce3fd",
          200: "#99c7fb",
          300: "#66aaf9",
          400: "#338ef7",
          500: "#006FEE",
          600: "#005bc4",
          700: "#004493",
          800: "#002e62",
          900: "#001731",
        },
        // .. rest of the colors
      },
    },
  },
};
```

> Common colors replace some of the [TailwindCSS](https://tailwindcss.com/docs/customizing-colors) default colors.
> See the [Removing Common Colors](#removing-common-colors) section to disable this feature.

You can also import the common colors in your JavaScript files.

```js
import {commonColors} from "@nextui-org/theme";

console.log(commonColors.white); // #FFFFFF
console.log(commonColors.blue[500]); // #006FEE
```

### Semantic Colors

Semantic colors automatically change with the theme, they help communicate meaning and reinforce the brand.

We recommend adding a palette that ranges from `50` to `900`. You can use tools like [Eva Design System](https://colors.eva.design/),
[Smart Watch](https://smart-swatch.netlify.app/), [Palette](https://palettte.app/) or [Color Box](https://colorbox.io/) to generate your palette.

> Semantic colors are placed inside the `nextui` plugin options and not inside the TailwindCSS `theme` object.

```js {7-13,19-25,32-36}
module.exports = {
  plugins: [
    nextui({
      themes: {
        light: {
          colors: {
            background: "#FFFFFF", // or DEFAULT
            foreground: "#11181C", // or 50 to 900 DEFAULT
            primary: {
              //... 50 to 900
              foreground: "#FFFFFF",
              DEFAULT: "#006FEE",
            },
            // ... rest of the colors
          },
        },
        dark: {
          colors: {
            background: "#000000", // or DEFAULT
            foreground: "#ECEDEE", // or 50 to 900 DEFAULT
            primary: {
              //... 50 to 900
              foreground: "#FFFFFF",
              DEFAULT: "#006FEE",
            },
          },
          // ... rest of the colors
        },
        mytheme: {
          // custom theme
          extend: "dark",
          colors: {
            primary: {
              DEFAULT: "#BEF264",
              foreground: "#000000",
            },
            focus: "#BEF264",
          },
        },
      },
    }),
  ],
};
```

### Using Semantic Colors

Semantic colors are available to use everywhere in your preoject where you use colors,
like the text color utilities, border color utilities, background color utilities, and more.

```html
<div class="bg-primary-500 text-primary-50 rounded-small px-unit-2 py-unit-1">
  This is a primary color box
</div>
```

### Javascript Variables

You can also import the semantic colors in your JavaScript files.

```js
import {semanticColors} from "@nextui-org/theme";

console.log(semanticColors.dark.warning.DEFAULT); // #FFC107
console.log(semanticColors.light.primary.DEFAULT); // #006FEE
```

### CSS Variables

NextUI creates CSS variables using the format `--prefix-colorname-shade` for each semantic color. By
default the prefix is `nextui`, but you can change it with the `prefix` option.

```js {4}
module.exports = {
  plugins: [
    nextui({
      prefix: "myapp",
    }),
  ],
};
```

Then you can use the CSS variables in your CSS files.

```css
/* With default prefix */
.my-component {
  background-color: var(--nextui-primary-500);
}
/*  With custom prefix */
.my-component {
  background-color: var(--myapp-primary-500);
}
```
